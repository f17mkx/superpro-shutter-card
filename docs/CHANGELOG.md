- `✨ feat: v0.5 test harness - Vitest + Playwright + Rollup build pipeline` - the only HA shutter card with an actual CI green-bar. Added `package.json` with rollup/vitest/playwright/lit toolchain. Source moved from `dist/` to `src/superpro-shutter-card.js`; Rollup bundles it back to `dist/superpro-shutter-card.js` as a self-contained IIFE with Lit 3.x inlined (~170KB, no sibling files). Removed the old `dist/lit/` subfolder (now bundled). Pure-logic helpers `buildConfig`, `getUniqueKeysFromObjects`, `levelToColorKey`, `tiltAngleDeg`, `normalisePosition`, `boundary` extracted to module scope and exported for unit testing - the live `#buildConfig` private method now delegates. 60 Vitest unit tests covering: preset+user merge order across all 5 presets, deprecated-keyword migration (`invert_percentage` → `invert_percentage_cover`, `invert_open_close` → `invert_open_close_ui`), removed-keyword migration (`title_position` → `name_position`), unknown-keyword warnings, ICONCOLORS 5-band quantiser used by battery+signal indicators, position normalisation + invertCover/invertUi composition, tilt linear-interpolation + negation, custom element registration. 4 Playwright smoke tests against `tests/e2e/fixtures/ha-stub.html` (a hand-rolled hass shim - no docker HA needed): card renders without throwing, click→`cover.open_cover`, `doOnclick(set_cover_position, 75)`→`cover.set_cover_position` with correct payload, `prefers-color-scheme: dark` triggers the v0.2 brightness/contrast/saturate filter on slat assets (regression guard for the v0.2 signature feature). CI workflow `.github/workflows/test.yaml` runs `npm ci` → build → dist-drift check (`scripts/check-dist-drift.mjs` byte-compares committed dist against rebuild) → unit tests → e2e tests on every PR. Existing `validate.yaml` (HACS validator) untouched and runs in parallel. VERSION bumped `v0.4.0` → `v0.5.0` in src + manifest.

- `✨ feat: v0.4 i18n (DE/EN/FR/ES) - third signature feature` - card now follows HA's UI language reactively across the four locales covering >80% of HA's active user base; other locales gracefully fall back to English.
  - **Two-tier translation strategy**: card-specific strings live in a new inline `TRANSLATIONS` const (just below `LOCALIZE_TEXT`) covering `battery`/`signal` aria-prefixes, `locked` (passive-mode lock), `fully_open`/`fully_closed`/`partially_open`/`partially_closed` (partial-open button + position-grid labels), and the `tilt` position-text prefix. Cover states (`open`/`closed`/`opening`/`closing`/`unavailable`) and action verbs (`open_cover`/`close_cover`/`stop_cover`/`*_tilt`) keep delegating to HA core's `hass.localize()` via the existing `LOCALIZE_TEXT` map - those already ship in every HA UI language so we deliberately don't duplicate them.
  - **`getT(key)` helper** on `shutterCfg`: reads `hass.locale.language` (first 2 chars, lowercased), falls back to `en` for any unmapped locale, falls back to the raw key as final defensive fallback. Reactive: `updateHass(hass)` is called from `SuperproShutterCardNew.shouldUpdate()` whenever LitElement signals a `hass` property change, so user toggling HA → Profile → Language flips every card immediately, no reload.
  - **`formatPercent(value)` helper**: uses `Intl.NumberFormat(locale, { style: 'percent', maximumFractionDigits: 0 })`. Real visible difference - `en`/`es` render `55%` compact, `de`/`fr` render `55 %` with non-breaking space, which is what native users expect. Wrapped in try/catch with `value + '%'` fallback for theoretical Intl-less environments. Wired into `positionToText()` (alwaysPercentage path + partial-open path), `computePositionText()` (offset-active parenthetical + tilt suffix), the partial-open button label, and all 4 partial entries in the position-grid `labels` dict.
  - **Render-site swaps**: 6 hardcoded English literals replaced with `cfg.getT(...)` / `cfg.formatPercent(...)` calls - battery aria-label, signal aria-label, lock-icon aria-label, partial-open button label (×2: label + aria-label), position-grid labels (×6: 0..5), and the position-text composite. Console warnings (`Unknown keyword: [...]`, `Deprecated: [...]`, `Removed: [...]`) stay English on purpose - they're developer-facing config diagnostics, not end-user UI.
  - **Anti-scope honoured**: preset config tokens (`awning`, `curtain`, `roller-shutter`, `shade`, `blind`) deliberately NOT translated (breaking change for existing YAML); no JSON-file loader (single-file inline dict matches v0.4's zero-build shape, JSON split lands with v0.5's build pipeline); ARIA labels from v0.3 are now part of the i18n target and translate alongside everything else.
  - **README**: added v0.4 i18n section with the two-tier strategy table, locale-aware percent example, and a "Contributing a new locale" subsection (4 numbered steps + screenshot expectation). v0.4 row in the roadmap table flipped to ✅; v0.5 absorbs the `+ performance` scope that was originally bundled with v0.4.
  - VERSION bumped `v0.3.0` → `v0.4.0` in `dist/superpro-shutter-card.js` and `manifest.json`. Still zero-build; v0.5 owns the pipeline.

- `✨ feat: v0.3 accessibility + clean console - second signature feature` - first HA shutter card to ship a deliberate a11y pass against the upstream baseline.
  - **ARIA**: every `ha-icon-button` (Up/Stop/Down/Partial/Tilt-Up/Tilt-Down/position-grid) now carries an explicit `aria-label` mirroring the localised tooltip, and decorative `<ha-icon>` children are marked `aria-hidden="true"` so screen readers don't double-read the glyph plus the label. Battery and signal pills wrap their icon+text combo in `role="img"` with a composed `aria-label="Battery 83%"` style. The shutter container itself is `role="group" aria-label="{friendly name}"`. Lock indicator (passive mode) becomes `role="img" aria-label="locked"`.
  - **Keyboard focus**: added `:focus-visible` outline on `.${ESC_CLASS_BUTTONS} ha-icon-button` and the tilt-button cluster, honouring `var(--focus-ring-color, currentColor)` with 2px solid + 2px offset, scoped to `:focus-visible` so mouse clicks stay clean. Tab order naturally flows up → stop → down → partial → tilt cluster (the picture-area picker is a non-interactive div, correctly absent from tab order).
  - **Live region**: new visually-hidden `<div role="status" aria-live="polite" aria-atomic="true">` rendered as the last child of every shutter element, populated via a debounced `_scheduleAnnounce()` (500ms) that skips during user-drag / user-drag-tilt phases so VoiceOver/NVDA hear "{name}: {position text}" only when the shutter settles. Pending timer is cleared in `disconnectedCallback` to prevent setTimeout-on-detached-element leaks. Added new screen-reader-only utility class `.esc-sr-only` (clip-rect technique).
  - **Console cleanup**: introduced top-level `const SILENCE = true` flag. The two upstream noise sources `Could not find grid container` and `Could not find div.card to set CSS variables` are now gated behind `else if (!SILENCE)` and routed through `console_log` instead of `console.warn` - both fire harmlessly inside HA's panel-view dashboards which legitimately have no `.container` or `div.card` ancestor. Also routed through `console_log`: `'ShutterCard  .. no content ..'` (×2, transient empty config during HA startup), `'Passive mode, no action'` (intentional config behaviour), `Failed to load image: ... using default` (graceful fallback already happened), and `haEntity: Entity [...] not found` (already surfaces as visible "unavailable" badge). Kept as real warnings/errors: service-not-available errors, image-dimension load failures, message-manager warnings, and the version banner `console.info` at file bottom.
  - VERSION bumped `v0.2.0` → `v0.3.0` in `dist/superpro-shutter-card.js` and `manifest.json`. Still zero-build (v0.5 owns the pipeline).
